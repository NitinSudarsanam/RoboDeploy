### D4. Scaffolder CLI ‚Äî `robodeploy/scaffold.py` (NEW, ~500 lines)

```bash
robodeploy scaffold task --name kitchen_pick --template pick_place --output examples/tasks/kitchen_pick.py
robodeploy scaffold policy --name custom_reach --template reach_dsl --output examples/policies/custom_reach.yaml
robodeploy scaffold robot --name myarm --dof 6 --description-dir robodeploy/description/myarm/
robodeploy scaffold sensor --name pressure --backend mujoco --output robodeploy/sensors/pressure/sim/mujoco_pressure.py
robodeploy scaffold preset --name myrobot_pick --robot myarm --backend mujoco --task pick_place --output examples/config/presets.yaml
robodeploy scaffold example --name my_demo --preset kuka_pick_mujoco --output examples/my_demo/run.py
```

Implementation: jinja2 templates in `robodeploy/templates/`. Each generates skeleton file with TODO comments + docstring + `@register_*` decorator + example body.

Example: scaffolded task body (~25 lines, vs hand-written 86):
```python
"""KitchenPick task ‚Äî your description here."""
from robodeploy.core.registry import register_task
from robodeploy.tasks.templates import PickPlaceTemplate
from robodeploy.scene_builder import SceneBuilder

@register_task("kitchen_pick")
class KitchenPick(PickPlaceTemplate):
    source_name = "mug"
    target_name = "tray"
    reward_weights = {"ee_to_source": 0.4, "source_to_target": 1.0, "lift_bonus": 0.15}

    def scene_spec(self):
        return (SceneBuilder()
                .add_table(height=0.4)
                .add_mesh("mug", asset="mug.obj", pos=(0.55, 0, 0.41), mass=0.08)
                .add_target("tray", pos=(0.65, 0.2, 0.41))
                .validate(backend=self.backend_name)
                .build_spec())

    def language_instruction(self): return "Move the mug onto the tray."
```

### D5. Linter CLI ‚Äî `robodeploy/linter.py` (NEW, ~400 lines)

```bash
robodeploy lint task examples/tasks/kitchen_pick.py
robodeploy lint policy examples/policies/custom_reach.py
robodeploy lint preset examples/config/presets.yaml --check kuka_pick_mujoco
robodeploy lint all
```

Checks:
- `@register_task` / `@register_policy` decorator present.
- All abstract methods implemented (`scene_spec`, `reward_fn`, `success_fn`, etc.).
- `action_space` declared.
- Required obs_spec fields match sensor rig.
- Reward function signature: `reward_fn(self, obs, action) -> float`.
- No use of deprecated APIs (e.g., `backend.has_prop_contact` warning if outside backend).
- YAML reach DSL: phases have required fields, target names exist in scene.
- Preset: backend kwargs match backend signature.

Output: rich-formatted error/warning list with file:line + suggested fix.

## Acceptance Criteria

- [ ] New user runs `pip install robodeploy && robodeploy run-episode --preset dummy` and sees output in < 60 seconds.
- [ ] Tutorial 02 produces working custom task in <30 lines from scratch.
- [ ] `robodeploy scaffold task --name X --template pick_place` produces compilable file.
- [ ] `robodeploy scaffold policy --name X --template reach_dsl` produces working YAML + loader.
- [ ] `robodeploy lint task examples/tasks/pick_place.py` reports no issues; introducing a missing method triggers error.
- [ ] `robodeploy scene validate scene.yaml --backend mujoco` exits non-zero on bad scene.
- [ ] `robodeploy doctor` accurately reports installed dependencies.
- [ ] Every public class in `robodeploy/core/interfaces/` has API docstring.
- [ ] `docs/` builds via `mkdocs build` without warnings.
- [ ] GitHub Pages auto-deploy on `main` push.
- [ ] Migration guide covers each renamed/removed symbol in 0.1.x ‚Üí 0.2.x.

## Dependencies

- `jinja2` (scaffolder templates).
- `rich` (CLI output formatting).
- `mkdocs-material` + `pdoc` (docs build).
- `pyyaml` (preset/lint).

## Risks

- **Docs drift**: examples bit-rot. Mitigation: CI step runs `docs/tutorials/*.md` code snippets via `pytest --markdown-docs`.
- **Scaffolder noise**: generated files become outdated. Mitigation: template version field + `robodeploy lint` warns on outdated template.
- **API ref clutter**: pdoc shows everything. Mitigation: explicit `__all__` lists in modules.
